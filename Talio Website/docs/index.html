Introduction

Welcome to the Talio API documentation. Our RESTful API allows you to programmatically access and manage your workforce data, integrate with external systems, and build custom applications on top of Talio.

Base URL

All API requests should be made to: https://api.talio.app/v1

Key Features

  • RESTful JSON API
  • OAuth 2.0 Authentication
  • Real-time Webhooks
  • Comprehensive SDKs
  • Rate limiting with generous quotas

Quick Start

Get up and running with the Talio API in minutes. Follow these steps to make your first API call.

Step 1: Get Your API Key

Navigate to Settings â†’ API Keys in your Talio dashboard to generate a new API key.

Keep Your Keys Secure

Never expose your API keys in client-side code or public repositories. Always use environment variables.

Step 2: Make Your First Request

Here's a simple example to fetch your employee list:

cURL 
curl -X GET "https://api.talio.app/v1/employees" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
JavaScript 
const response = await fetch('https://api.talio.app/v1/employees', {
  method: 'GET',
  headers: {
    'Authorization': `Bearer ${process.env.TALIO_API_KEY}`,
    'Content-Type': 'application/json'
  }
});

const employees = await response.json();
console.log(employees);
Python 
import os
import requests

api_key = os.environ.get('TALIO_API_KEY')
headers = {
    'Authorization': f'Bearer {api_key}',
    'Content-Type': 'application/json'
}

response = requests.get(
    'https://api.talio.app/v1/employees',
    headers=headers
)

employees = response.json()
print(employees)

Authentication

Talio uses OAuth 2.0 for API authentication. All API requests must include a valid access token in the Authorization header.

Bearer Token

Include your API key in all requests:

HTTP Header 
Authorization: Bearer YOUR_API_KEY

API Overview

The Talio API provides access to all major features of the platform. Here's a quick overview of available endpoints:

GET /employees

Retrieve a list of all employees in your organization.

POST /attendance/check-in

Record an employee check-in with GPS coordinates.

GET /leave/balance/:employeeId

Get leave balance for a specific employee.

POST /payroll/generate

Generate payroll for a specific month.

Rate Limits

To ensure fair usage and platform stability, API requests are rate-limited:

  • Free Plan: 100 requests per minute
  • Professional: 1,000 requests per minute
  • Enterprise: 10,000 requests per minute (customizable)

Rate limit headers are included in all responses:

Response Headers 
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200

Error Handling

The API uses standard HTTP status codes to indicate success or failure:

  • 200 - Success
  • 400 - Bad Request (invalid parameters)
  • 401 - Unauthorized (missing or invalid API key)
  • 403 - Forbidden (insufficient permissions)
  • 404 - Not Found
  • 429 - Rate Limit Exceeded
  • 500 - Internal Server Error
